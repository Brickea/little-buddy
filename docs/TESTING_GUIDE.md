# iPhone 测试指南 (Testing Guide)

本指南帮助你在 iPhone 真机上运行和测试 Little Buddy 原型。

---

## 前置要求

- **Mac 电脑**（运行 macOS 14+）
- **Xcode 15+**（从 Mac App Store 安装）
- **iPhone**（iOS 17+）
- **Apple ID**（免费即可，无需付费开发者账号）
- **USB 数据线**

---

## 第一步：生成 Xcode 项目

### 方法 A：使用 XcodeGen（推荐 ⭐）

```bash
# 安装 XcodeGen（只需一次）
brew install xcodegen

# 进入项目目录
cd LittleBuddy

# 生成 Xcode 项目
xcodegen generate

# 打开项目
open LittleBuddy.xcodeproj
```

### 方法 B：手动创建 Xcode 项目

1. 打开 Xcode
2. **File → New → Project**
3. 选择 **iOS → App**
4. 填写信息：
   - Product Name: `LittleBuddy`
   - Interface: **SwiftUI**
   - Language: **Swift**
   - Minimum Deployments: **iOS 17.0**
5. 选择保存位置（不要保存在仓库内，避免冲突）
6. **删除** Xcode 自动生成的以下文件：
   - `ContentView.swift`
   - `LittleBuddyApp.swift`
7. 右键点击项目导航器中的 `LittleBuddy` 文件夹 → **Add Files to "LittleBuddy"**
8. 导航到仓库的 `LittleBuddy/Sources/LittleBuddy` 文件夹
9. 确保选择 **"Create groups"**（不是 "Create folder references"）
10. 勾选 **LittleBuddy target**
11. 点击 **Add**

---

## 第二步：配置 API Key（AI 角色生成）

Little Buddy 已集成 ChatAnywhere LLM API，可以使用真实 AI 来根据描述生成角色。

### 配置方式

在 Xcode 中为 App 配置环境变量：

1. 点击 Xcode 顶部的 **Scheme**（`LittleBuddy` 旁边的运行目标）
2. 选择 **Edit Scheme...**
3. 在左侧选择 **Run**
4. 点击 **Arguments** 标签
5. 在 **Environment Variables** 区域添加：
   - Name: `CHAT_ANYWHERE_API_KEY`
   - Value: `你的 ChatAnywhere API Key`
6. 确保勾选了这个变量

> 💡 如果没有 API Key，App 仍然可以正常运行——角色会使用本地关键词匹配生成。获取免费 API Key：https://api.chatanywhere.org/

### 验证 API Key 是否生效

创建角色时：
- **有 AI**：角色名字更有创意，技能描述更丰富，属性分配更贴合描述
- **无 AI（回退模式）**：角色名字取自描述前几个字，技能从预设池生成

---

## 第三步：配置签名

1. 在 Xcode 左侧导航器中，点击项目文件（蓝色图标）
2. 选择 **LittleBuddy** target
3. 点击 **Signing & Capabilities** 标签
4. 勾选 **Automatically manage signing**
5. **Team** 下拉菜单中选择你的 Apple ID
   - 如果没有，点击 **Add Account...** 登录你的 Apple ID
6. **Bundle Identifier** 修改为唯一值，例如：`com.yourname.littlebuddy`

> 💡 使用免费 Apple ID 时，App 在设备上的有效期为 7 天，过期后需要重新安装。

---

## 第四步：在 iPhone 上运行

1. 用 USB 数据线连接 iPhone 和 Mac
2. 在 iPhone 上点击 **信任此电脑**
3. 在 Xcode 顶部的设备选择器中，选择你的 iPhone
4. 点击 **▶️ 运行按钮**（或按 `Cmd + R`）
5. 首次运行时，iPhone 上需要信任开发者：
   - 打开 **设置 → 通用 → VPN与设备管理**
   - 找到你的开发者账号
   - 点击 **信任**
6. 回到 Xcode，再次点击运行

---

## 第五步：测试场景

### 测试 1：创建角色 ✨（AI 生成）

1. 打开 App，你会看到首页
2. 点击 **"创建新伙伴 ✨"**
3. 在文本框中输入角色描述，建议尝试：
   - `一个会喷火的蓝色机器人，拳头是铁锤` → 应该生成火系机器人
   - `超级快的风之鸟，会飞` → 应该生成风系动物
   - `坚硬的岩石巨人，防御很强` → 应该生成土系角色，防御较高
   - `可爱的水精灵` → 应该生成水系角色
   - `一个神秘的小怪兽` → 应该生成无属性怪兽
4. 点击 **"生成角色 ✨"**

> 💡 **LLM AI 模式**：如果已配置 ChatAnywhere API Key，角色将由真实 AI（GPT-4o-mini）生成，角色名字、技能、属性会更贴合你的描述。如果网络不可用或 API Key 未配置，会自动回退到本地关键词匹配生成。

5. **检查要点**：
   - [ ] 角色名称显示正确？
   - [ ] 元素属性是否与描述匹配？（火、水、风、土）
   - [ ] 属性值是否合理？（HP + 攻击 + 防御 + 速度 = 100）
   - [ ] 技能数量 2–4 个？
   - [ ] 技能元素是否匹配角色元素？
   - [ ] 角色头像 Emoji 和颜色是否正确？
6. 点击 **"保存小伙伴 💾"**
7. 点击 **"返回首页 🏠"**
8. 确认角色出现在首页的 "我的小伙伴" 列表中

### 测试 2：快速对战 ⚔️

1. 在首页点击 **"快速对战 ⚔️"**
2. 系统会自动生成两个角色并开始对战
3. **对战流程**：
   - 查看谁先手（取决于速度值）
   - 轮到你时，选择一个技能
   - 观察伤害数值和 HP 变化
   - 对手自动选择技能攻击
   - 重复直到一方 HP 归零
4. **检查要点**：
   - [ ] 先手判定是否正确？（速度高的先行动）
   - [ ] 伤害数值是否合理？
   - [ ] HP 条动画是否流畅？
   - [ ] 元素克制 💥 提示是否正确？（火克风、水克火、风克土、土克水）
   - [ ] 技能冷却是否生效？（使用后显示冷却回合数）
   - [ ] 胜利/失败界面是否正常显示？
   - [ ] 点击 "返回首页" 是否正常返回？

### 测试 3：角色对战

1. 先通过 "创建新伙伴" 创建至少一个角色
2. 在首页 "我的小伙伴" 列表中，点击角色旁边的 **"对战 ⚔️"** 按钮
3. 与随机生成的 AI 对手进行对战
4. **检查要点**：
   - [ ] 是否正确使用了你创建的角色？
   - [ ] AI 对手是否随机生成且合理？
   - [ ] 对战结束后是否能正常返回？

### 测试 4：元素克制验证

1. 创建一个 **火系角色**（描述中包含"火"）
2. 创建一个 **风系角色**（描述中包含"风"）
3. 用火系角色对战，如果遇到风系对手：
   - 你的火系技能应该触发 💥 元素克制，伤害更高
4. **克制关系参考**：
   - 🔥 火 → 🌪️ 风
   - 💧 水 → 🔥 火
   - 🌪️ 风 → 🪨 土
   - 🪨 土 → 💧 水

### 测试 5：多角色管理

1. 创建 3 个不同元素的角色
2. 确认所有角色都出现在首页列表中
3. 分别用不同角色进行对战
4. **检查要点**：
   - [ ] 所有角色是否正确显示？
   - [ ] 每个角色的对战功能是否正常？

### 测试 6：双人对战（Pass-and-Play）👫

1. 在首页点击 **"双人对战 👫"**
2. **玩家 1 选择角色**：
   - 点击 **"随机生成角色 🎲"** 快速生成，或从已有角色列表中选择
3. 看到 **"把设备交给玩家 2！"** 提示后，将设备递给第二位玩家
4. 点击 **"玩家 2 准备好了 ✋"**
5. **玩家 2 选择角色**（同上操作）
6. 两个角色显示在 "准备开战！" 界面，点击 **"开始对战！🔥"**
7. **对战流程**：
   - 每回合开始前会显示 **"请将设备交给玩家 X"** 的传屏提示
   - 点击 "准备好了" 后显示技能选择界面
   - 当前玩家选择技能后，观察伤害和 HP 变化
   - 自动切换到下一位玩家（再次显示传屏提示）
   - 重复直到一方 HP 归零
8. **检查要点**：
   - [ ] 传屏提示是否正确显示？（防止偷看对方技能）
   - [ ] 每位玩家只能在自己回合选择技能？
   - [ ] 伤害计算和元素克制是否正常？
   - [ ] 技能冷却是否正确？
   - [ ] 胜利界面是否显示正确的获胜玩家？
   - [ ] 点击 "返回首页" 是否正常返回？

### 测试 7：SpriteKit 角色渲染 🎨

1. 创建一个角色（或进行快速对战）
2. 观察角色的 SpriteKit 渲染效果：
   - 对战界面中角色形象应显示为几何图形组合的火柴人/色块风格
   - 角色预览界面也应显示 SpriteKit 渲染的角色形象
3. **检查要点**：
   - [ ] 角色形象是否根据体型（robot/monster/animal/humanoid/dragon）显示不同造型？
   - [ ] 元素颜色是否正确反映角色元素？
   - [ ] 不同大小的角色（small/medium/large）是否有视觉差异？

## 常见问题

### Q: Xcode 报错 "Untrusted Developer"
A: 在 iPhone 上进入 设置 → 通用 → VPN与设备管理 → 信任开发者证书

### Q: 编译报错 "No signing certificate"
A: 在 Signing & Capabilities 中重新选择你的 Apple ID，确保勾选了 Automatically manage signing

### Q: App 图标显示为空白
A: 这是 Phase 0 原型的正常情况，暂未添加 App 图标

### Q: 关闭 App 后角色消失了
A: Phase 0 使用内存存储，这是预期行为。Phase 1 将添加持久化存储

### Q: 创建角色时提示 "LLM 生成失败"
A: 这是正常的——如果 API Key 未配置或网络不通，会自动回退到本地关键词匹配生成角色。如果需要 AI 生成，请在 Xcode Scheme 中配置 `CHAT_ANYWHERE_API_KEY` 环境变量（参考第二步）
